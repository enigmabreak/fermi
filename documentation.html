Fermi Studio Documentation

Welcome to Fermi Studio, the secure coding environment designed for academic integrity.

For Students: How to Use the Application

  • Workspace Configuration: Upon launch, you must enter your 11-digit University Roll Number. This initializes your secure session.
  • Saving Work: Your C files are automatically saved to your secure workspace. To compile and run, click the "Run" button in the bottom left corner.
  • External Files: If you open a C file that was not handwritten in this session, the editor will flag it with a reddish tab border and log the event.
  • Focus Tracking: Do not minimize the window or switch to other applications (like web browsers). The IDE actively tracks focus loss.
  • USB Lockdown: Do not insert USB drives during a session. Doing so will immediately lock the screen until the device is removed, and the incident will be logged.
  • Paste Restrictions: Pasting code is strictly prohibited to prevent plagiarism.

For Professors: Supermode (Admin Mode)

To assist students or debug issues without triggering integrity violations, invigilators can use the built-in Supermode.

  • Activation: Type admin into the Roll Number field during startup.
  • Capabilities: Supermode automatically bypasses all paste restrictions and disables window-switch monitoring, allowing you to freely copy/paste code and navigate the OS.
  • Admin Panel: Entering Supermode automatically opens the Admin Panel, where you can view, manage, and cryptographically verify all submitted student Integrity Reports.
  • Stealth: To maintain clean logs, Supermode completely disables Integrity Report generation for the session.

For Professors: Scoring & Integrity Systems

Fermi Studio operates a deterministic risk scoring model to quantify potential academic misconduct.

Cryptographic Report Integrity

When a session concludes, the IDE generates an HTML integrity report. To prevent tampering, the entire raw event log is serialized and hashed using SHA-256. This hash is embedded at the bottom of the report.

If a student attempts to modify the HTML file to remove a logged event or alter the timestamps, the hash will automatically invalidate when verified against the payload.
